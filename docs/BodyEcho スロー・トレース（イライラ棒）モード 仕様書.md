# **BodyEcho 追加機能：スロー・トレース（イライラ棒）モード 仕様書**

本ドキュメントは、AR骨格トラッキングアプリケーション「BodyEcho」における新しいゲームモード「スロー・トレース（イライラ棒）モード」の追加仕様を定義する。本モードは、ユーザーの力加減（スピードコントロール）および身体の可動域に対する客観的理解（マッピング）を深めることを目的とする。

## **1\. 機能概要**

暗い画面上に浮かび上がるコース（淡く発光する道）に沿って、ゆっくりと移動する目標（光の球）に対し、ユーザーが指定された手首のノードを重ね合わせ、一定の速度を保ちながら追従（トレース）するゲームモード。コースからはみ出したり、光の球から遅れたりすると、マイルドなエラーフィードバックが発生する仕組みである。

## **2\. 目的とアプローチ**

| 課題 | 本モードによるアプローチ   |
| :---- | :---- |
| **力加減（コントロール）の難しさ** | 急なオン・オフの運動ではなく、一定の筋緊張を保ちながらゆっくり動かし続ける「スローコントロール」の感覚を視覚フィードバックで養う。 |
| **身体可動域（リーチ）の不鮮明さ** | 自分の腕が空間のどこまで届くのか、ゲーム内で自然にキャリブレーションを行い、届かない場所への目標出現による挫折を防ぐ。 |

## **3\. システム・ゲームロジック仕様**

### **3.1. 動的キャリブレーション（肩幅基準相対座標）**

ユーザーの体格やカメラとの距離に関わらず、常に適切な難易度を維持するための仕様。

* MediaPipe Poseより取得する「左肩（LEFT\_SHOULDER）」と「右肩（RIGHT\_SHOULDER）」の2点間距離を基準値（1 Unit）とする。  
* コース全体の描画可能領域および光の球の最大移動半径を、この基準値の1.2倍〜1.5倍の範囲（安全可動域）に動的に制限する。

### **3.2. 追従・判定ロジック（軽量衝突検知）**

パフォーマンス（超低遅延）を維持するため、ピクセル単位の検知ではなく数式（ピタゴラスの定理）による距離判定を行う。

* 毎フレーム、目標（光の球）の中心座標 $(X\_t, Y\_t)$ と、追従する利き手の手首ノード座標 $(X\_w, Y\_w)$ の直線距離 $D$ を計算する：  
  $$D \= \\sqrt{(X\_t \- X\_w)^2 \+ (Y\_t \- Y\_w)^2}$$  
* 距離 $D$ が、あらかじめ設定された許容半径 $R$ （画面高の数％、または肩幅基準の相対値）以下であれば「キープ（成功）」、上回れば「はみ出し（エラー）」と判定する。

### **3.3. コース・軌道生成パターン**

初期フェーズでは以下の2パターンの軌道をサポートする。

1. **直線トレース（水平・垂直）:** 身体の正中線を意識しやすいよう、胸の前を横切る、または上下に真っ直ぐ動かす基本コース。  
2. **緩やかなカーブ（サイン波・円弧）:** 関節を滑らかに連動させるための、サイン波や円弧状のコース。

## **4\. UI/UX・演出仕様**

### **4.1. 視覚演出（空気と余白の維持）**

* 背景は完全なダークトーン（黒または単色グレー）を維持し、コースのガイド線は淡い輝度のシアン/マゼンタで描画する。  
* 手首ノードが光の球の中に正しく収まっている間は、球体が優しく脈動（パルス）するように発光する。  
* はみ出し（エラー）が発生した際は、過度な点滅を避け、光の球が一時的に半透明の霧のように霧散する、またはノード周辺に小さな火花（パーティクル）が短時間散るに留める（罰ではなく、発見としてのデザイン）。

### **4.2. 聴覚フィードバック仕様 (audioSynth.ts 連携)**

* **キープ中（成功時）:** 心地よい空間的な広がりを持つ正弦波（Sine波）の和音が、一定の音量で鳴り続ける（運動の継続性を音で保証する）。  
* **はみ出し時（エラー時）:** 水滴が静かに落ちるような減衰の早いピチカート音、または丸みのある矩形波による「コッ」という短い遮断音を鳴らし、緊張感を与えすぎずに状態の変化を伝える。

## **5\. 開発における注意・配慮事項**

* **認知疲労の防止:** 視覚優位性を利用した身体制御は脳への負荷が高いため、1プレイの制限時間は最長で60秒〜90秒程度とし、強制的にインターバルを挟む設計にする。  
* **音と描画の完全同期:** Canvasの描画処理落ちがWeb Audio APIの発音タイミングに影響を及ぼさないよう、発音イベントはタイムスタンプを用いて優先的にスケジューリングする。
