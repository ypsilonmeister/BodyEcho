# **ボディイメージ・ミラー アプリケーション仕様書**

本ドキュメントは、身体感覚（固有受容覚）の認識や力加減に難しさを抱える児童を支援するための、AR骨格トラッキングを活用したデジタルミラーアプリケーション「ボディイメージ・ミラー」の基本仕様を定義する。

## **1\. 開発背景と目的**

「体育館に鏡があるけれど、自分の動きがわからない」という課題に対し、視覚的なアプローチを用いて身体認知を補完することを目的とする。現実のカメラ映像から不要な視覚ノイズを排除し、関節と骨格の連動という「構造」のみを抽象化してフィードバックすることで、脳の処理負荷を下げ、自身の身体境界線や運動軌跡を直感的にマッピングできるようにする。

## **2\. システム概要**

スマートフォン、タブレット、および大型テレビ（PC接続）のマルチデバイスに対応したWebアプリケーション。カメラ映像からリアルタイムで骨格検知（Pose Estimation）を行い、単色背景上に抽象化された骨格の線を描画する。

## **3\. 基本要件 (Core Requirements)**

| 項目 | 詳細仕様   |
| :---- | :---- |
| **リアルタイム性** | 身体感覚の書き換え（マッピング）を正確に行うため、最低30fps（推奨60fps）の超低遅延描画を維持する。 |
| **エッジ推論** | ネットワーク遅延によるズレを排除するため、すべての姿勢推定処理はサーバーを介さず、ローカルデバイス上（オンデバイス）で完結させる。 |
| **ミラーリング** | 実際の鏡と同一の体験を提供するため、カメラ映像を左右反転（鏡面表現）した状態で処理・描画する。 |

## **4\. 視覚・UIデザイン仕様 (Visual & UX Design)**

### **4.1. 画面構成**

* **完全な抽象化:** カメラの生映像は一切描画せず、背景は単色（黒またはダークグレー）で塗りつぶす。  
* **空気と余白の重視:** 操作ボタンや説明テキストなどの視覚的ノイズを完全に削ぎ落とし、広い暗い背景（余白）のなかに骨格の線だけが浮かび上がる静謐なトーンを徹底する。

### **4.2. 骨格の描画ルール**

* **ノード（関節）:** 主要な関節（肩、肘、手首、股関節、膝、足首）を強調された点として描画。  
* **エッジ（骨）:** 関節同士を繋ぐ線として描画。  
* **左右のカラーコーディング:** 身体の正中線の交差を意識しやすくするため、右半身（例：シアン）と左半身（例：マゼンタ）で明確に色分けを行う。

## **5\. マルチデバイス対応仕様 (Multi-Device Support)**

### **5.1. レスポンシブ・相対スケーリング**

10インチ前後のタブレットから、50インチ以上の大型テレビまで、多様な画面サイズで同一のスケール感を提供するための設計。

* **動的描画座標:** 姿勢推定エンジンから取得した0.0〜1.0の正規化座標データを、常に実行画面の幅と高さ（Viewport）に乗算して描画位置を決定する。  
* **相対サイズ定義:** 線の太さやノードの直径は、画面の縦幅に対するパーセンテージ（例：画面高の1%）として動的に計算し、大画面でも細すぎたり、小画面で潰れたりするのを防ぐ。

### **5.2. ハンズフリー・インタラクション**

デバイスから離れた位置での利用を想定し、画面タッチやマウスによる操作を不要にする。

* **オートキャリブレーション:** カメラの画角内に全身（最低限、両肩から両膝までのノード）が3秒間収まったことを検知すると、自動的にトラッキングを開始する。  
* **ジェスチャーリセット:** トラッキングの乱れやエラーが発生した場合は、画面から離れた状態で「両手を頭の上に上げる（Yのポーズ）」など特定のジェスチャーを検知させることで、システムを自動的にリセット・再認識させる。

## **6\. 技術スタック (Technical Stack)**

* **推論エンジン:** Google MediaPipe Pose (JavaScript / WebAssembly)  
* **描画エンジン:** HTML5 \<canvas\> API (2D Context)  
* **フロントエンド環境:** Vanilla JavaScript または軽量なフロントエンドフレームワークによるWebブラウザ完結型構築

## **7\. 今後の拡張インタラクション案 (Future Scope)**

* **軌跡（トレイル）エフェクト:** 手首と足首のノードに、動かした後に数秒間光の帯（DNAの螺旋構造を模したパーティクルなど）が残る残像エフェクトを実装し、運動のスピードと空間的な軌道を視覚化する。  
* **角度・達成フィードバック:** 「肘が180度まっすぐ伸びた」などの条件を満たした際に、骨格の線が発光したり効果音が鳴ったりする、直感的な目標達成フィードバックの追加。
